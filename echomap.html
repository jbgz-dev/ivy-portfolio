Core Experience

从真实地点出发，形成“听见回声 - 生成任务 - 导航探索 - 留下新回声”的闭环。

EchoMap 不只是地图标注，也不是普通旅游攻略。它把位置、附近 POI、用户记忆和 AI 任务生成串成一次可行动的空间体验。

进入地点定位到真实空间，创建探索会话
空间感知识别地址、天气、附近 POI 与场景氛围
听见回声展示附近用户留下的故事、光影与现实建议
生成任务AI 结合选中回声和真实地点生成行动任务
步行导航调用百度地图路线，引导用户前往任务点
现场完成到达后拍摄、录音或文字记录发现
留下回声把本次探索沉淀成后来者可听见的新记忆

AI Architecture

5 个 AI Agent 协作，把地图信息转成可执行探索。

后端基于 Supabase Edge Functions 拆分 Agent 能力，每个 Agent 输出结构化结果，并配置 fallback 保证演示链路稳定。

spatial-awareness

空间感知

接入百度地图逆地理编码、天气与附近 POI，推断当前地点的 sceneType、lightLevel、spaceMood 和任务提示。

echo-memory

回声池管理

读取附近真实回声，按距离、类型和推荐理由组织成可选择的回声卡片，不足时生成兜底内容。

task-generator

任务生成

基于用户选中的回声、空间感知结果和附近真实 POI 坐标，生成可导航、可到点、可完成的探索任务。

echo-collect

内容归档

处理用户现场提交的文字、图片或录音，判断回声类型并沉淀到数据库，成为后来者的探索素材。

result-summary

结果总结

汇总本次探索路径、听过的回声、完成的任务和留下的内容，生成可分享的个人探索结果页。
